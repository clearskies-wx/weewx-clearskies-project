# Pinned Items — Things to Revisit

Items that came up during review and need to be addressed but aren't blocking the current phase.

---

| # | Item | Context | Opened |
|---|------|---------|--------|
| 1 | **Chart migration should be in the wizard, not a CLI tool** | The `clearskies-migrate-charts` CLI converts Belchertown `graphs.conf` → Clear Skies `charts.conf`. Operators shouldn't need to run a command-line tool for this — the import step (step 2) should offer to upload or locate `graphs.conf` and do the conversion automatically. | 2026-07-03 |
| 2 | **Wizard step 4 — full rework needed** | Multiple issues: (a) SQLite is weewx's default DB but the wizard only shows MySQL fields — needs to detect DB type from `weewx.conf` and show the right form; (b) help content should remind the operator which DB type was detected and that fields are pre-filled; (c) help should explain how to recover DB credentials if forgotten (check `weewx.conf` under `[DatabaseTypes] [[MySQL]]`, or `mysql -u root -p` to reset). Sort out the SQLite support first, then write the help content to match. | 2026-07-03 |
| 3 | **Help button discoverability — intro animation** | Operators won't know the help button exists unless we point it out. Add a one-time animation on the first wizard step (pulse, bounce, or tooltip arrow) that draws attention to the help icon. Should only play once (sessionStorage flag) so it doesn't annoy on subsequent steps. | 2026-07-03 |
| 4 | **Xweather display name — drop "(Vaisala)"** | `display_name="Xweather (Vaisala)"` appears in 4 API provider modules (forecast, radar, alerts, AQI). Their product branding is just "Xweather" — the "(Vaisala)" parenthetical isn't how they refer to themselves. Change `display_name` to "Xweather" site-wide. Keep `attribution_text` as "Vaisala Xweather" — this is the required plain-text attribution per their terms. 29 occurrences across 8 files in the API repo. | 2026-07-03 |
| 5 | **Move Units step (7) to right after Column Mapping (5)** | Step 5 (Column Mapping) already deals with units — source unit detection, unit badges, overrides. Step 7 (Display Units) sets how those same measurements are shown to visitors. These should be adjacent. Currently step 6 (Station Identity) sits between them, breaking the flow. Reorder to: 5 Column Mapping → 6 Display Units → 7 Station Identity. Requires renumbering step templates, route handlers, progress bar, and all "Step X of 15" heading strings. | 2026-07-03 |
| 6 | **Stop auto-selecting providers ("Selected for your location")** | The wizard pre-selects a provider per domain via a `recommendations` dict and shows a "Selected for your location" badge. Operators pick their own providers. | 2026-07-03 |
| 7 | **Remove OWM from AQI — it is REMOVED, not deprecated** | `aqi/openweathermap.py` still exists with a deprecation warning. It needs to be fully deleted, not just deprecated. | 9 | **Remove LibreWxR geographic bounds from wizard** | `step_providers.html` lines 226–242 have an optional bounds input for LibreWxR (`south,west,north,east` CSV). | 10 | **AQI Regional Scale — clarify which providers support it** | The wizard shows an AQI regional scale selector. Need to verify: is this only for Xweather, or do other AQI providers also return region-specific scales? The help text should make clear which providers this setting applies to, and it should only appear when a provider that supports it is selected. | 2026-07-03 |
| 11 | **Step 9 help — comprehensive per-provider details with collapsible sections** | Current help is too thin. | 12 | **Weather alerts — expand provider options** | Currently NWS (US, keyless) is the primary alerts provider. Xweather offers alerts globally and OWM offers alerts too. Investigate whether Open-Meteo provides alerts. Ensure operators outside the US have a path to alerts. | 2026-07-03 |

| 17 | **Seismic API enhancements** | Multiple improvements needed: (a) **Epicenter distance** — compute distance from operator's station to each earthquake epicenter and include in the response; (b) **Modified Mercalli Intensity (MMI)** — add estimated shaking intensity, which is more meaningful to non-scientists than magnitude alone; (c) **Local unit conversion** — convert distance and depth to the operator's preferred units (km or miles) instead of always returning metric; (d) **Depth unit override** — add depth to the wizard/admin unit selectors like we do for other measurements, so operators can display depth in km or miles. | 2026-07-04 |
| 18 | **Volcanic activity monitoring** | **Primary source: Smithsonian GVP** — global volcano database (1,400+ active + dormant) for map overlay, plus **daily activity reports** (Mon–Fri, joint with USGS) for current eruptions and elevated activity worldwide. Daily cadence is sufficient for a weather site. **Secondary (US only): USGS Volcano APIs** — real-time alert levels (NORMAL/ADVISORY/WATCH/WARNING) via `getMonitoredVolcanoes` JSON endpoint, useful for US operators near volcanoes. Approach: Smithsonian daily reports for global coverage now, add local provider sources later if needed. | 2026-07-04 |
