A3 — Icon System Expansion · 32 Hero Icons

All hero weather glyphs after the Icon System Overhaul (Phase 1): the 11 existing Material-Symbols-based glyphs from weather-icon-glyphs.tsx plus 21 new glyphs — mostly-cloudy day/night, drizzle, wintry mix, 6 combined sky+precipitation icons, and 13 atmosphere-condition icons (haze, smoke, dust) across cloud-cover tiers. Every SVG below has its own scoped <defs> block (i<n>- prefixed gradient IDs) since this is a static HTML file, not React with useId(). WMO code labels are sourced from docs/planning/briefs/FORECAST-ICON-SELECTION-BRIEF.md §2.2 and §5.4 (proposed extension codes for smoke/dust are marked with *).

1 · Sky conditions

Clear, partly cloudy, mostly cloudy (NEW — cloud-cover tier between partly cloudy and overcast), and overcast. Mostly-cloudy reuses the Sunny/Bedtime path scaled to 55% and tucked into the top-right corner behind the dominant overcast cloud — "barely visible" celestial body.

S1 Sky conditions 7 icons · 5 existing + 2 new

2 · Precipitation

Rain, snow, and thunderstorm are existing glyphs. Drizzle (NEW) swaps the rain cloud's angled streaks for small round dots — visually lighter than rain. Wintry mix (NEW) combines 2 rain streaks (left) with 3 snow dots (right) under one cloud — serves freezing rain, sleet, and rain/snow mix per the icon-selection brief §5.2.

S2 Precipitation 5 icons · 3 existing + 2 new

3 · Combined sky + precipitation (all NEW)

For the 20–50% PoP band where the sky is still partly visible (icon-selection brief §5.3). The partly-cloudy cloud+sun/moon group is scaled to 80% and shifted up to leave room for rain streaks and/or snow dots layered below.

S3 Combined sky + precipitation 6 icons · all new

4 · Atmosphere conditions — haze, smoke, dust

Three different composition techniques per icon-selection brief §5.5: haze uses the existing cutout technique (clip sky element at y=16.5, haze stripes below) extended to partly-cloudy and overcast; smoke uses an overlay technique (smoke bubbles superimposed on the intact base icon); dust uses a standalone technique (dust particles dominate, sun/moon/cloud reduced to a small background element) since Meteocons treats dust as visually incompatible with cloud overlay.
